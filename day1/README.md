### 과제 4

나눌 때 분모가 0이 되는 경우, 거듭 제곱에서 지수가 정수형이 아닐 경우에 예외처리를 해주었다.

"계속하려면 아무 키나 누르십시오 . . ."라는 문구가 있어서 직관적으로 봤을 때 키를 누르는 즉시 반응해야 할 것 같아 구글링을 해서 _getch()라는 함수를 찾았다. 입력값이 1 ~ 255라고 하여 _getch()를 받는 변수를 -1로 설정해두고 while문 안에 _getch()값이 입력되기 전까지 빈 코드를 반복하게 하여 지연 효과를 주었다.

추가 수정(아래 코드 블럭에 기재)

+ 지수가 음수인 경우에  i < b 조건을 i > b로 바꾸고 answer *= 1/a로 바꿔주면 지수가 1보다 작은 경우도 표현할 수 있다.

```c
case '^': {
  int int_b = (int)b;
  if (b - int_b == 0)
  {
      answer = 1;
      if (b > 0)
      {
      for (int i = 0; i < b; i++)
          answer *= a;
      }
      else // 지수가 음수인 경우
      {
      for (int i = 0; i > b; i++)
          answer *= 1/a;
      }
      
      break;
  }
  else
      printf("거듭 제곱의 지수는 정수만 입력 가능");
  return;
}

```

### 과제 7

팩토리얼이 수가 조금만 커지면 오버플로우가 일어나서 결과값 자료형을 unsigned long long으로 해두었다.

> 그래도 숫자가 너무 커지니 별 차이가 없는 것 같다...

+ 확통을 안해봐서 그런지 해당 내용 쓰면서 다시 보니까 오류가 있었다.

순열, 조합과 중복 순열, 중복 조합의 조건이 다르다.

순열, 조합 ( 0 < r =< n, r과 n은 정수 )

중복 순열, 중복 조합 ( 0 < n, 0 < r, r과 n은 정수 )

그래도 찾아보니까 순열, 조합의 조건이 더 강한 조건이어서 사용자 경험 측면을 무시(?)한다면 4가지 상황 모두를 충족시키는 n, r값에 대해서만 경우의 수를 출력한다고 치고 메세지만 추가하면 될 것 같다.

```c
    if (r > n)
    {
        printf("r > n");
        printf("\n 순열, 조합 조건이 성립하지 않습니다."); // 중복 순열, 중복 조합의 경우 성립하지만 코드 종료
        return;
    }

    if (r < 1)
    {
        printf("\n0 < r =< n");
        return;
    }
```
